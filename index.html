THE RECON FRAMEWORK

By team HEXCREW

Application that provides seamless indoor navigation using WIFI signals of that location and available sensors (Gyroscope, Accelerometer, Magnetic Field) of any basic smartphones. No additional infrastructure (extra beacon) or no known WIFI network is required (only the signal strength is measured, so the user’s smartphone need not be connected).


We implemented this in two aspects


THE RECON FRAMEWORK

 Organisations/developers can use our algorithms and modules to create custom IPS applications for their organisations based on the requirements.

PLOTSENZ

 Mobile application (built using the Recon Framework) that provides indoor navigation and LBS (Location Based Services)

IPS is not widespread like GPS as the requirement differs with case and infrastructure. By providing IPS development tools (that do not require external infrastructure) as a framework organisations/developers can build IPS applications easily


We implemented a real-time error correction ML Algorithm to filter noise in sensor data and improvised existing IPS technology to scale and Plotsenz acts as an overlay to existing GPS applications which seamlessly switches to IPS mode on entering indoors. Plotsenz universal IPS map is similar to google maps, where public contributors can scan and add locations to the map.


Problem Statement Title


Seamless indoor-outdoor navigation


Problem Statement


One of the major uses of Navigation based applications is Location-Based Services (LBS) which constitutes a significant portion of the navigation application usage. The availability of location-sensitive smartphones, with wireless broadband high-speed connections, has made low-cost applications available to a huge audience and there exists a lot of innovation in designing new types of technology and applications using these devices. Typically, it is seen that outdoor positioning is generally GPS/GNSS based. However, indoor positioning is generally based on WLAN technology. The primary reason is that GPS generally does not work for indoor applications. Uninterrupted good accuracy positioning needs thinking in the fusion of multiple technologies (Smart Phone Sensors, Wi-Fi, BLE beacons, GNSS, Dead Reckoning, etc.) to deliver a seamless outdoor and indoor positioning experience. To design/develop new ideas and mobile apps which make use of these technologies (and even other signals of opportunity) to provide seamless indoor-outdoor positioning and provide innovative location-based services.


The following table summarises our innovation and improvisations to IPS technologies in the context of The RECON FRAMEWORK and the app PLOTSENZ (android version)


Method Existing Problems Our Innovation
GNSS N/A The framework uses Global Navigation Satellite System data in the model creation algorithms for calibrating the in-app coordinate systems
Dead Reckoning Sensors - Noise and user’s casual hand movement causes False Steps ML-based algorithm for onboard real-time error correction
WIFI WIFI Repeaters: Same SSID so errors in computing location using the WIFI mapping methods The framework’s mapping algorithm fuses GPS data with the WIFI map, (Generally, wifi repeaters are separated by distances that GPS is accurate enough to differentiate)
WIFI Signal Strength Variations Between Devices (different signal strength values even at the same location) - due to differences in antenna hardware The Modules in the framework computes a special matrix, that is analogous to a structured ratio of signal strengths. Can be used to compute locations accurately even when WIFI signal strength differs.
WIFI Mobile Hotspots - recorded when mapping, might not be available during navigation. The mapping module can differentiate mobile AP from ACTUAL WIFI
Magnetic Field External Magnetic Field - when on move, changing environmental magnetic field creates errors in the angle computation Our modules can identify the influence of external magnetic field in the angle computation and calibrates automatically with the new magnetic field
Visualisation No general Visualisation /Analysis tools for spatial movement data We developed our own set of tools that can create path maps from spatial movement data and can be used for analysis.
Map Editor /Creator No tool available for creating indoor maps We developed an online map creator which can directly import raw spatial movement data recorded using the framework modules.

Introduction:


With current GPS Technology seamless navigation is possible for a person who has a smartphone. We can able to track down our location, wherever on Earth by GPS.


Consider a person who gets to attend an important examination. But his exam center is located in a remote place. GPS tracks down in spite of the remote location. But the real problem arises here. Once he gets there, he finds many campuses and infrastructure. When he needs to locate his exam hall, GPS is of no use. Even though GPS tracks the hardest location, it is limited to a certain point of location and it could not give the exact indoor location. A panic scenario starts as he couldn’t able to find his examination hall at the right time. Indoor accuracy is the greatest drawback of GPS.


This problem can be overcome by IPS ( Indoor Positioning System). Even though IPS can be seen /developed, it is not widespread due to its limitations like external hardware requirements and smartphone sensor abilities. Mostly they are limited to a particular organisation and cannot be used by common people.


IPS can be seen in supermarkets, Malls, and companies. It works only in that particular area which gains profits and advantages to those companies, whereas the common man gets no use of it as he could not use it whenever he desires. Common people are being affected by this in various situations. Take a real situation, where a patient is admitted to a hospital, which lacks nurses. When he is in need of a nurse, the person who is accompanying the patient tries to find the nurse but she is nowhere to be found. As the person doesn’t know in which room she would be in.

Even in Factories which has different sectors, accidents may take place anywhere in those sectors. But without IPS, members who need to save the people couldn’t be able track down the exact position of the incident. Alike this there are so many places like airports, movie theatres, parking garages, stadiums, etc.. which are in need of IPS.


To overcome all these problems our project gives a valid solution only by using smartphones and their available sensors. As each and every building has a WIFI signal, with the help of it and our designed app, we would be able to provide Indoor Positioning System. As soon as the person opens his phone to track his room/class, our app provides the signal which eventually led to the desired place. This works just like GPS right? Yes Of course. Along with the magnificent work of both GPS and IPS, we can track any location within a span of time.

Due to this students might easily get to their examination hall without any stress. Advanced nurse calling systems could locate the nearest nurse. Several people can be saved from accidents in factories.

Our main ambition is to provide an easy tracker which can be used in day-to-day life by any person who has a smartphone and can track the location just like GPS
